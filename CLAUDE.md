## Architecture

The project has three layers: **web app** (Flask), **ML pipeline** (scikit-learn), and **data collection** (Spotify/Deezer/Genius/iTunes APIs).

### Mood Classification

`src/mood_classifier.py` uses the energy/valence quadrant model:
- High energy + high valence = **happy**
- High energy + low valence = **aggressive**
- Low energy + high valence = **chill**
- Low energy + low valence = **sad**

Center zone (both 0.35–0.65) defaults to chill unless danceability > 0.7.

### API Fallback Chain

Spotify restricted audio features for apps registered after Nov 2024. The analyze route in `web/app.py` handles this:

1. Try Spotify `audio-features` endpoint
2. If 403 → fetch real BPM from **Deezer** free API (`/search` + `/track/{id}`)
3. Estimate remaining features via `src/data_processing/add_missing_columns.py` (genre/popularity/year heuristics)
4. Mark tracks as `estimated: True` so the UI labels them

### Song Story Sources (priority order)

1. **Genius API** — `/api/search/multi` + `/api/songs/{id}?text_format=plain` for real About section descriptions
2. **Handwritten stories** — `SONG_STORIES` dict in `web/app.py` (~20 iconic songs)
3. **Wikipedia API** — `/w/api.php` search + `/api/rest_v1/page/summary/` with artist name validation
4. Generic fallback message

### Web App Data Flow

```
/playlists → /loading/<id> (shows animated loading page)
           → /analyze/<id> (fetches tracks, features, stories, previews)
```

`analyze_playlist_data()` uses `ThreadPoolExecutor(max_workers=8)` to fetch Genius + iTunes in parallel. Limited to first 50 tracks (`MAX_DETAILED_TRACKS`) for performance.

iTunes Search API provides 30-second preview URLs (free, no auth).

### ML Pipeline

`src/pipelines/ml_pipeline.py` — `SpotifyMLPipeline` class:
- Trains 4 classifiers (NaiveBayes, GradientBoosting, DecisionTree, RandomForest)
- Runs KMeans clustering (5 clusters)
- Uses 8 features: tempo, danceability, energy, valence, acousticness, speechiness, instrumentalness, loudness
- Results saved to `data/results/results_TIMESTAMP.json`

### Key Constants

- `REDIRECT_URI = 'http://127.0.0.1:8888/callback'` — must match Spotify dashboard
- `MAX_DETAILED_TRACKS = 50` — Genius/iTunes lookup limit
- Playlist track cap: 200 (in analyze route)
- Mood colors: happy=#F59E0B, sad=#6366F1, aggressive=#EF4444, chill=#10B981

## Credentials

`.env` file (gitignored) with `SPOTIFY_CLIENT_ID` and `SPOTIFY_CLIENT_SECRET`. The redirect URI `http://127.0.0.1:8888/callback` must be registered in the Spotify Developer Dashboard. Genius, Deezer, iTunes, and Wikipedia APIs are free and need no keys.

## Sample Dataset

`data/sample/sample_playlist.csv` — 76 tracks with pre-computed audio features and mood labels. Used by demo mode and CLI pipeline default. Generated by `data/sample/generate_sample.py`.
